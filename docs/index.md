## 这个站点适合放什么

- 学习路线、阶段任务和交付物
- AI 基础概念与系统知识地图
- Prompt、RAG、Tool Calling 等构建模块
- Agent、工作流和工程化设计
- 可以快速运行和修改的最小实战代码
- 可复用模板、复盘机制和参考资料

## 维护原则

1. 文档优先写在 `docs/`，可运行代码优先放在 `examples/`。
2. 新内容先判断属于 `learning / foundations / components / systems / resources` 哪一层，再开始写。
3. 每篇文档至少回答四个问题：这是什么、为什么重要、怎么用、容易错在哪。
4. 收藏资料之前先判断是否值得复习，不把这里做成“链接堆”。

## 现在就可以做的事

- 从 [从这里开始](/guide/start-here) 了解整个知识库的使用方式
- 从 [学习路径总览](/learning/) 确认学习顺序和阶段目标
- 从 [核心原理](/foundations/) 搭建自己的知识地图
- 从 [实战项目](/practice/) 找一个最小 demo 跑起来
