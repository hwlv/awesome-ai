# 站点路线图

这个路线图的重点不是“把所有 AI 知识都写完”，而是让这个仓库持续变得更有用。

## 阶段一：站点可用

- 建好 VitePress 文档站点
- 接上 GitHub Pages 自动部署
- 补齐主导航、侧边栏、搜索和基础页面
- 让仓库具备最小阅读体验

## 阶段二：知识骨架成型

优先补这些主题：

- AI 基础概念总览
- Prompt 基础与模板
- Embeddings、向量检索、RAG
- Tool calling 与函数式工具接入
- Agent 基础、模式和边界

完成后，这个站点就已经具备“能回看、能复习、能分享”的价值。

## 阶段三：示例与复盘

- 每个核心主题补一个最小 demo
- 开始记录真实踩坑
- 建立每周复盘和待补清单
- 把参考资料从“收藏”转成“已吸收 / 待消化 / 需回看”

## 阶段四：专题深化

后面可以逐步新增专题：

- RAG 工程实践
- Agent memory 与 planning
- 多 agent 协作
- 模型选型与成本控制
- Prompt / Agent 评估

## 内容质量标准

你后续新增文档时，建议尽量满足这几个标准：

1. 不只写定义，要写场景和边界。
2. 不只放链接，要写为什么值得看。
3. 不只贴代码，要写这段代码说明了什么。
4. 不追求全面，优先保证可复习和可复用。
