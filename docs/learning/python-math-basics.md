# 阶段 1：Python 与数学基础

这一阶段的目标不是“把数学全学完”，而是把后面学机器学习真正会用到的底座补平。

## 阶段目标

- 熟练使用 Python 做基础脚本、数据处理和环境管理
- 掌握 NumPy、Pandas 的常见操作
- 复习线性代数、微积分、概率统计的核心概念
- 理解监督学习、无监督学习、训练集和评估指标这些基础术语

## 核心任务

1. 配好 Python 学习环境，熟悉虚拟环境、依赖安装和 Notebook。
2. 回顾 Python 语法、函数、类、异常处理、模块与包。
3. 用 NumPy 和 Pandas 做一次完整的数据读取、清洗、分析和可视化。
4. 整理向量、矩阵、导数、梯度、期望、方差这些概念的公式卡片和例题。
5. 理解过拟合、欠拟合、训练集、验证集、测试集这些后面会反复出现的概念。

## 建议实践

- 做 2 到 3 个 Kaggle 入门题或 Hugging Face 入门实验
- 用公开数据集完成一次小型 EDA
- 写一个简单 API，把数据处理结果返回成 JSON

## 建议交付物

- 至少 3 个结构清晰的 Notebook 或脚本
- 一份数学概念笔记，覆盖至少 10 个核心概念
- 一篇阶段总结，回答“我现在能独立完成什么”

## 推荐资料

- 《Python Crash Course》前 12 章
- Real Python、Kaggle Learn 的 Python 和 Pandas 入门内容
- 3Blue1Brown 的线性代数系列
- Andrew Ng 机器学习课程前几周或《Hands-On ML》第 1 章

## 配套工程建议

如果要在当前仓库放这个阶段的代码，建议拆成下面这样：

```text
examples/learning/stage-1-python-math/
├─ README.md
├─ scripts/
├─ notebooks/
└─ outputs/
```

这样比把说明和任务清单混在代码目录里更容易维护：

- `README.md` 负责介绍怎么运行
- `scripts/` 放一次性演示代码
- `notebooks/` 放探索式实验
- `outputs/` 放图表和导出结果

## 常见误区

- 一开始就去追模型和框架名字，结果基础数据处理能力没跟上
- 只看数学定义，不把公式变成代码和图表
- 做了练习但没有留下结论，后面回头几乎等于重学

## 完成标准

- 能独立写一个数据处理脚本并解释输出
- 能说清楚常见机器学习概念之间的关系
- 能看懂并改动一个最小的 Python 数据分析项目
