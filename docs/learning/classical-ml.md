# 阶段 2：经典机器学习

这一阶段的重点是把“会跑模型”推进到“会做完整建模流程”。

## 阶段目标

- 熟悉数据获取、清洗、特征工程、建模、评估和迭代的完整链路
- 理解常见算法的适用场景和优缺点
- 学会用 scikit-learn Pipeline 组织可复用流程
- 能把最优模型封装成 API

## 核心任务

1. 选一个真实数据集，完成缺失值处理、异常检测和特征编码。
2. 在同一数据集上训练至少 3 种模型并比较结果。
3. 实践 `Pipeline`、交叉验证和超参数搜索。
4. 用特征重要性或 SHAP/LIME 做基础解释。
5. 把最佳模型做成可调用接口。
6. 写一份技术报告，说明问题定义、结果和上线风险。

## 建议实践

- 用 `House Prices`、`Credit Card Default` 这类公开案例做端到端演练
- 配一个最小 Dashboard，真正消费模型结果
- 每次调参都记录原因和结果，不要只保留最好的一次

## 建议交付物

- 一个从数据到模型的完整项目
- 一份模型评估结果和实验记录
- 一个可交互测试的 API Demo
- 一篇阶段总结，说明模型效果和下一阶段准备度

## 推荐资料

- 《Hands-On Machine Learning》第 2 到 9 章
- scikit-learn 官方教程
- Made With ML 的建模部分
- StatQuest 和 Kaggle Learn 的经典 ML 系列

## 配套工程建议

这一阶段更适合显式拆出数据、模型和接口：

```text
examples/learning/stage-2-classical-ml/
├─ README.md
├─ data/
├─ src/
├─ models/
└─ outputs/
```

目录职责建议：

- `src/`：数据流水线、训练脚本、API 入口
- `models/`：训练输出的模型文件
- `outputs/`：指标、图表、日志

## 常见误区

- 只盯最终指标，不记录特征工程和调参过程
- 模型能跑就算结束，没有解释为什么这个模型更合适
- API 只是包一层壳，没有考虑请求校验和错误处理

## 完成标准

- 能独立完成一次经典 ML 项目闭环
- 能解释不同模型在同一任务上的差异
- 能把训练结果稳定暴露为可消费接口
