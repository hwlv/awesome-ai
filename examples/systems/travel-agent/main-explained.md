# 旅行助手 Agent — main.py 代码逻辑详解

> 看完这篇文档，你不用再逐行读代码，就能完全理解这个 Agent 是怎么运转的。

---

## 一句话总结

`main.py` 实现了一个 **ReAct Agent 循环**：用户提问 → LLM 思考并决定行动 → 程序执行工具 → 把结果喂回 LLM → LLM 继续思考……直到给出最终答案。

---

## 分步拆解

### 第 0 步：启动与初始化

| 做了什么 | 对应代码 | 说明 |
|---------|---------|------|
| 加载环境变量 | `load_dotenv()` | 从 `.env` 文件读取 API Key、模型名等配置 |
| 创建 LLM 客户端 | `build_llm_client()` | 内部实例化 `OpenAICompatibleClient`，支持 OpenAI / DeepSeek 等兼容接口 |
| 获取用户输入 | `get_user_prompt()` | 命令行有参数就用参数，没有就用默认的"查北京天气+推荐景点" |
| 初始化 prompt 历史 | `prompt_history = [...]` | 这个列表就是 Agent 的"记忆"，后续每轮都会往里追加内容 |
| 注册工具 | `available_tools = {...}` | 把字符串名称映射到真实的 Python 函数，Agent 靠名称匹配来调用 |

### 第 1 步：调用 LLM

```python
full_prompt = "\n".join(prompt_history)
llm_output = llm.generate(full_prompt, system_prompt=AGENT_SYSTEM_PROMPT)
```

把 **所有历史记录** 拼成一个大字符串，连同系统提示词一起发给 LLM。

系统提示词告诉 LLM：
- 你是旅行助手
- 你有 `get_weather` 和 `get_attraction` 两个工具
- 每次回复必须是 `Thought: ... Action: ...` 格式

### 第 2 步：提取 Action 块

```python
llm_output = extract_action_block(llm_output)
```

LLM 有时候会"抢跑"，一口气输出好几步 Thought/Action/Observation。这个函数用正则只截取 **第一组**，保证每轮只执行一个动作。

### 第 3 步：解析 Action

三种情况：

| 情况 | 判断条件 | 处理方式 |
|------|---------|---------|
| **没有 Action** | 正则匹配不到 `Action: ...` | 生成错误 Observation，提示 LLM 修正格式，进入下一轮 |
| **Finish** | Action 以 `Finish` 开头 | 提取方括号内的最终答案，打印并退出程序 |
| **工具调用** | Action 是 `func(arg="val")` 格式 | 进入第 4 步 |

### 第 4 步：执行工具

```python
tool_name, kwargs = parse_action(action_str)
observation = available_tools[tool_name](**kwargs)
```

`parse_action` 用正则把 `get_weather(city="北京")` 拆成：
- `tool_name` = `"get_weather"`
- `kwargs` = `{"city": "北京"}`

然后通过工具注册表找到对应函数，用 `**kwargs` 展开参数调用。

### 第 5 步：追加 Observation，回到第 1 步

```python
prompt_history.append(f"Observation: {observation}")
```

工具返回的结果被包装成 `Observation: ...` 格式，追加到历史。下一轮 LLM 就能看到这个结果，继续推理。

## 关键设计思想

### 1. ReAct 模式（Reasoning + Acting）

这是当前 AI Agent 最主流的设计模式之一。核心思想是让 LLM **交替进行推理和行动**：

```
Thought → Action → Observation → Thought → Action → Observation → ... → Finish
```

相比让 LLM 一步到位给答案，ReAct 的优势在于：
- LLM 可以 **分步推理**，降低出错概率
- 每一步都可以 **调用外部工具** 获取实时信息
- 整个推理过程 **可追踪、可调试**

### 2. Prompt 历史 = Agent 的记忆

`prompt_history` 列表是整个 Agent 的核心数据结构。每轮循环都会往里追加两条记录：
1. LLM 的输出（Thought + Action）
2. 工具的返回（Observation）

下一轮把完整历史拼接后发给 LLM，LLM 就能"记住"之前做了什么、得到了什么结果。

### 3. 工具注册表 = 字符串到函数的映射

```python
available_tools = {
    "get_weather": get_weather,
    "get_attraction": get_attraction,
}
```

这是一个简洁的"插件系统"。要新增工具，只需：
1. 写一个新的 Python 函数
2. 在字典里注册一行
3. 在系统提示词里告诉 LLM 这个工具的用法

### 4. 安全阀：MAX_STEPS

`MAX_STEPS = 5` 限制了最大循环次数。如果 LLM 一直不输出 Finish（比如陷入死循环或格式错误），程序会在 5 轮后强制退出，避免无限消耗 API 调用。

## 模块依赖关系

```
main.py
  ├── openai_compatible_client.py   ← LLM 调用封装（支持 OpenAI 兼容接口）
  ├── query_weather.py              ← 工具：调用 wttr.in 查天气
  ├── search_attraction.py          ← 工具：调用 Tavily 搜景点
  └── system_prompt.py              ← 系统提示词（定义 Agent 角色和输出格式）
```

## 快速运行

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置 .env 文件（至少需要 LLM_API_KEY 和 LLM_MODEL_ID）

# 3. 使用默认提示语运行
python main.py

# 4. 或者自定义提问
python main.py 帮我查上海的天气然后推荐景点
```
